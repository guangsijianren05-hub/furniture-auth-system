# セキュリティドキュメント

本システムは、お客様の個人情報を扱うため、最高レベルのセキュリティ対策を実装しています。

## 🔒 実装されているセキュリティ機能

### 1. 認証セキュリティ

#### パスワードポリシー
- **最低12文字必須**
- **複雑性要件:**
  - 英大文字を1文字以上
  - 英小文字を1文字以上
  - 数字を1文字以上
  - 記号を1文字以上
- **よくあるパスワードをブロック**

#### ログイン試行制限
- 5回連続失敗で30分間アカウントロック
- IPアドレスベースの追跡
- 自動ロック解除（30分後）

#### セッション管理
- 30分間操作がないと自動ログアウト
- アクティビティ監視（マウス、キーボード、クリック）
- セッションタイムアウトの通知

### 2. 通信セキュリティ

#### HTTPS強制
- すべての通信をSSL/TLSで暗号化
- HTTP接続は自動的にHTTPSにリダイレクト
- ローカル開発環境（localhost）を除く

### 3. データ保護

#### 個人情報マスキング
実装されているマスキング機能：

| データ種類 | 元データ | マスキング後 |
|-----------|---------|-------------|
| メールアドレス | kento.879301@gmail.com | ke***@gmail.com |
| 電話番号 | 090-1234-5678 | 090-****-5678 |
| 住所 | 東京都渋谷区〇〇 | 東京都*** |
| 顧客名 | 山田太郎 | 山田* |

**注意:** マスキングは画面表示のみです。データベースには完全な情報が保存されます。

#### データベースセキュリティ
- Firebase Security Rulesによるアクセス制御
- 権限ベースの読み書き制限
- サーバーサイドでの検証

### 4. XSS/CSRF対策

#### XSS（クロスサイトスクリプティング）対策
- すべてのユーザー入力をエスケープ
- HTMLタグの無害化
- Reactの自動エスケープ機能

#### CSRF（クロスサイトリクエストフォージェリ）対策
- Firebase Authenticationのトークンベース認証
- リクエストごとの検証

### 5. 監査ログ

#### 記録される情報
すべての重要な操作を記録：

| アクション | 記録内容 |
|-----------|---------|
| ログイン | 成功/失敗、IPアドレス、タイムスタンプ |
| ログアウト | ユーザー、タイムスタンプ |
| ユーザー作成 | 作成者、新規ユーザー情報、権限 |
| ユーザー削除 | 削除者、削除されたユーザー |
| 買取依頼更新 | 変更者、変更内容、変更前後の値 |
| データエクスポート | エクスポート者、形式、タイムスタンプ |
| 不正アクセス試行 | ユーザー、試行したアクション |

#### 監査ログの保存場所
Firestore Database: `audit_logs` コレクション

### 6. エラーハンドリング

#### ユーザーフレンドリーなエラーメッセージ
- 技術的な詳細を隠蔽
- 具体的な対処方法を提示
- リトライ機能の提供

#### エラーログ
- すべてのエラーを記録
- スタックトレース保存
- コンテキスト情報付き

### 7. ネットワークエラー対策

#### オフライン対応
- ネットワーク状態の監視
- オフライン時の適切な通知
- オンライン復帰時の自動再試行

#### リトライロジック
- ネットワークエラー時の自動リトライ（最大3回）
- 指数バックオフ（待機時間を徐々に延長）

## 🛡️ セキュリティベストプラクティス

### 管理者向け

#### 1. 強力なパスワードを使用
```
✅ 良い例: P@ssw0rd!2026#Secure
❌ 悪い例: password123
```

#### 2. 定期的なパスワード変更
- 3ヶ月ごとに変更を推奨

#### 3. アカウント管理
- 退職者のアカウントは即座に無効化
- 不要な権限は付与しない（最小権限の原則）

#### 4. 監査ログの定期確認
- 月1回は監査ログを確認
- 不審なアクセスがないかチェック

### スタッフ向け

#### 1. ログアウトの徹底
- 離席時は必ずログアウト
- ブラウザを閉じてもセッションが残る場合あり

#### 2. 画面共有時の注意
- 個人情報が表示されている画面を共有しない
- Zoom/Teams での画面共有時は注意

#### 3. 公共Wi-Fi使用時の注意
- 公共Wi-Fiでの利用は推奨しません
- VPN使用を検討

## 🚨 インシデント対応

### 不正アクセスの疑いがある場合

1. **即座にログアウト**
2. **管理者に報告**
3. **パスワード変更**
4. **監査ログを確認**

### アカウントがロックされた場合

1. **30分待つ**（自動ロック解除）
2. **それでもログインできない場合:**
   - 管理者に連絡
   - アカウントのリセットを依頼

### データ漏洩の疑いがある場合

1. **即座に管理者に報告**
2. **影響範囲の特定**
3. **監査ログの確認**
4. **必要に応じて顧客への通知**

## 🔐 Firebase Security Rules

本システムで使用しているセキュリティルール:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 認証済みユーザーのみアクセス可能
    function isAuthenticated() {
      return request.auth != null;
    }
    
    // ユーザーのロールを取得
    function getUserRole() {
      return get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role;
    }
    
    // マスター権限チェック
    function isMaster() {
      return isAuthenticated() && getUserRole() == 'master';
    }
    
    // 管理者以上の権限チェック
    function isAdminOrAbove() {
      return isAuthenticated() && getUserRole() in ['master', 'admin'];
    }
    
    // usersコレクション
    match /users/{userId} {
      allow read: if isAuthenticated();
      allow create, update, delete: if isMaster();
    }
    
    // purchasesコレクション
    match /purchases/{purchaseId} {
      allow read: if isAuthenticated();
      allow create: if isAuthenticated();
      allow update: if isAdminOrAbove();
      allow delete: if isAdminOrAbove();
    }
    
    // audit_logsコレクション
    match /audit_logs/{logId} {
      allow read: if isMaster();
      allow create: if isAuthenticated();
      allow update, delete: if false; // 監査ログは削除・編集不可
    }
  }
}
```

## 📱 デバイス別セキュリティ

### PC
- ✅ ウイルス対策ソフトを常に最新に

## 🔍 脆弱性報告

1. **報告先:** システム管理者
2. **報告内容:**
   - 問題の詳細
   - 再現手順
   - 影響範囲

   - 24時間以内に確認
   - 緊急度に応じて即座に修正
